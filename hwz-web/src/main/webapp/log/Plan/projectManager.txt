1.项目BUG管理流程
	测试发现bug --> 禅道提BUG -->  给项目负责人  -->  由负责人分配BUG   -->  解决完后提交BUG  -->   测试人员close

2.版本不统一问题
	问题来源：测试，预发布，正式环境的代码包是直接从每个开发人员本地打包上去。当某一个人没有在本地更新其他人员提交的代码，然后代码发布。
	
	解决方案：

								（合并代码）		      （合并代码）
	    本地文件 	    -- >    svn测试版本          -->     svn预发布版本       -->      svn正式版本
		|					|					|					|
	   本地开发调试	                测试服务器打包发布	                     预发布服务器打包发布	正式服务器打包发布

	通过合并代码保证 测试，预发布，正式版本一致。
